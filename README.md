# 딥러닝을 활용한 EEG 졸음 탐지 분류 성능 개선 연구

학부 시절 동서대학교 소프트웨어학과 ICaN Lab에서 학부 연구생으로 진행했던 EEG 기반 졸음 탐지 연구입니다. 뇌파(EEG) 신호로 졸음 상태를 분류하고, 그 분류 성능을 딥러닝으로 끌어올리는 것을 목표로 삼았습니다.

이 저장소에는 연구 초반의 데이터 전처리 단계, 그중에서도 원본 측정 데이터를 모델 학습에 쓸 수 있는 형태로 재구성하는 작업이 담겨 있습니다. 분류 모델 자체는 아직 코드로 정리되지 않은 상태이며, 여기 올라온 것은 그 앞단을 다지는 과정입니다.

## 데이터

원본 데이터는 MATLAB `.mat` 형식의 EEG 측정 파일입니다. `scipy.io.loadmat`으로 불러와 다룹니다. 용량 문제로 실제 데이터(`/data`)는 저장소에 포함하지 않았습니다.

데이터는 두 종류의 라벨로 나뉩니다.

- `sb`, `sg` 두 라벨로 각각 11개 파일(`sb1`~`sb11`, `sg1`~`sg11`)
- 한 사람당 `sb`, `sg` 데이터를 짝지어 묶음

측정 신호의 구조는 다음과 같습니다.

- 전체 68개 채널 중 1~64번이 EEG, 65~66번이 EOG, 67~68번이 ECG입니다. 이 연구에서는 EEG 64채널만 사용합니다.
- 샘플링 레이트는 512Hz입니다.
- 이벤트 값이 `1`인 지점이 측정 시작 시점이며, 그 앞부분을 잘라내고 시작 시점부터의 신호만 취합니다.
- 한 번의 측정은 대략 30분 길이로, 시작 시점 기준 약 921,600개의 샘플이 됩니다.

## 데이터 재구성

핵심 작업은 길게 이어진 1차원 시계열을 모델이 다루기 좋은 3차원 배열로 재구성하는 것입니다.

1분 단위(`SR × 60` = 30,720 샘플)로 신호를 잘라 트라이얼로 만들고, 이를 쌓아 `[trials × channels × time]` 형태로 만듭니다. 30분 측정이라면 `[30 × 64 × 30720]`이 됩니다.

```
[trials × channels × time-series] = [30 × 64 × 30720]
```

`EEGData` 클래스가 한 파일을 받아 로드, 시작 시점 정렬, 1분 단위 슬라이싱까지를 한 번에 처리합니다. 객체를 만들 때 내부적으로 재구성이 실행되어, 결과인 `ProcessedData`를 바로 꺼내 쓸 수 있습니다.

이후 사람 단위로 `sb`와 `sg` 트라이얼을 위아래로 이어붙여 `[trials*2 × 64 × 30720]` 배열을 만들고, `person_dict`에 `p1`~`p11`로 저장합니다. 두 라벨의 트라이얼 수가 다를 경우 더 적은 쪽에 맞춰 잘라냅니다.

재구성 결과가 맞는지는 MATLAB으로 따로 만든 `reconstruction.mat`과 `np.array_equal`로 값을 비교해 검증했습니다.

## 저장소 구성

```
src/
  EEG-Research-Main.ipynb              # 메인 노트북: EEGData 클래스, 사람 단위 재구성
  Data-Reconstrion-Single-Data.ipynb  # 단일 파일 재구성 과정을 단계별로 풀어 쓴 노트북
  TestCodes.ipynb                      # 재구성 로직 실험·테스트 코드
img/
  matlab_data_view.png                 # MATLAB에서 본 원본 데이터 구조
```

`Data-Reconstrion-Single-Data.ipynb`는 한 파일을 두고 타입 확인부터 이벤트 분리, 슬라이싱까지 한 단계씩 따라가며 정리한 노트북입니다. 그 과정을 클래스로 묶어 여러 파일에 적용한 것이 메인 노트북입니다.
